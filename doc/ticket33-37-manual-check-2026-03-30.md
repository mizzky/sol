# チケット33-37 手動確認手順

作成日: 2026-03-30

## 事前準備
1. バックエンドを起動する
   - ワークスペースタスク: `Go: Air (Backend)`
2. フロントエンドを起動する
   - ワークスペースタスク: `Next.js: Dev (Frontend)`
3. ブラウザで `http://localhost:3000` を開く
4. 管理系機能を確認する場合は、管理者ユーザーでログインする

## 確認項目

### チケット33: 商品詳細ページ
1. トップページで任意の商品カードの「詳細を見る」を押す
2. `/products/{id}` に遷移することを確認する
3. 商品名、価格、カテゴリ、SKU、在庫数、説明が表示されることを確認する
4. 数量を変更して「カートに追加」を押し、完了メッセージが表示されることを確認する
5. ヘッダーのカート件数が更新されることを確認する

### チケット34, 35: 管理者商品管理
1. 管理者でログイン後、`/admin/products` を開く
2. 「新規商品登録」フォームでカテゴリ欄がプルダウンになっていることを確認する
3. 新しい商品を登録し、右側の一覧に反映されることを確認する
4. 一覧の「編集」を押し、商品名やカテゴリを変更して更新できることを確認する
5. 一覧の「削除」を押し、確認ダイアログ後に商品が消えることを確認する
6. 各商品の「公開ページを確認」から詳細ページへ移動できることを確認する

### チケット36: 管理者カテゴリ管理
1. `/admin/categories` を開く
2. カテゴリを新規作成し、一覧へ表示されることを確認する
3. 一覧の「編集」を押し、カテゴリ名または説明を更新できることを確認する
4. 一覧の「削除」を押し、確認ダイアログ後に削除できることを確認する
5. `/admin/products` に戻り、カテゴリプルダウンに変更内容が反映されていることを確認する

### チケット37: 管理者ユーザー権限変更
1. `/admin/users` を開く
2. 案内文に、ユーザーID入力ベースの暫定UIであることが表示されることを確認する
3. 不正なIDとして `0` を入力して送信し、バリデーションエラーが表示されることを確認する
4. 実在するユーザーIDを入力し、role を `admin` または `member` に切り替えて送信する
5. 成功メッセージと「直近の更新内容」に user_id と role が表示されることを確認する
6. 対象ユーザーで再ログインし、管理画面へのアクセス可否が変わることを確認する

## 補足
- 商品やカテゴリの登録に失敗する場合は、バックエンドのレスポンス内容とブラウザ開発者ツールの Network タブを確認する
- チケット37は一覧API未提供のため、対象ユーザーIDは別手段で確認してから操作する

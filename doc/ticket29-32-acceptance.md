### チケット30: カート画面のチェックアウト

- [x] [frontend/app/cart/page.tsx](frontend/app/cart/page.tsx) で「チェックアウトへ進む」押下時に注文作成が実行される
- [x] 注文作成成功時に `/orders` へ遷移する
- [x] 注文作成失敗時に以下のメッセージ分岐が動作する
  - 400: カートが空です
  - 401: 認証が必要です
  - 409: 在庫不足のため注文を作成できません
  - その他: 注文作成に失敗しました

確認手順（手動）:
- 400: 「カートを空にする」実行後に「チェックアウトへ進む」を押下
- 401: ブラウザのCookie/Storageから認証情報を削除してから「チェックアウトへ進む」を押下
- 409: 在庫を超える数量をカートに入れて「チェックアウトへ進む」を押下
- その他(5xx想定): 手動再現は難しいため、フロント単体テストで担保

- [x] ログイン状態でヘッダーに `Orders` リンクが表示され、注文履歴画面へ遷移できる

### チケット32: 注文キャンセル

- [x] `pending` ステータス注文にのみ「注文をキャンセル」ボタンが表示される
- [x] キャンセル成功後、注文一覧が再取得され、対象注文のステータスが `cancelled` になる
- [x] キャンセル失敗時に以下の分岐が表示される
  - 400: この注文はキャンセルできません
  - 404: 注文が見つかりません
  - その他: 注文キャンセルに失敗しました

確認手順（手動）:
- 400: 同一注文に対して二重キャンセル状態を作る必要があり、画面操作のみでは再現しづらい
- 404: UI導線上で存在しないIDを指定できないため、画面操作のみでは再現しづらい
- その他(5xx想定): 手動再現は難しいため、フロント単体テストで担保
- 本ドキュメントの手動確認では成功ケースを必須、失敗ケースはテスト確認を代替エビデンスとする

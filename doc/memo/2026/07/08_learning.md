### 躓いたポイントと解決策
- `DO $guard$ ... $guard$` の `guard` が、関数名や外部から呼べる名前のように見えて混乱した。
  - `guard` は PostgreSQL の dollar quote の区切り名であり、変数名・関数名・プロシージャ名ではないと整理した。
- `DO ' ... '` と `DO $$ ... $$` と `DO $guard$ ... $guard$` の関係が分かりにくかった。
  - `DO` は一回限りの PL/pgSQL ブロックを実行するための構文で、本文は文字列として渡す。シングルクォートだとエスケープが増えるため、dollar quote を使うと読みやすいと整理した。
- `DO` とストアドプロシージャの違いが曖昧だった。
  - `DO` は DB に保存されず、その場で一回だけ実行される無名ブロック。`CREATE PROCEDURE` や `CREATE FUNCTION` は DB に保存され、あとから名前で呼べる処理だと整理した。
- seed SQL の実装メモに何を書くべきか迷った。
  - 各 SQL の件数、ガード処理、`ON_ERROR_STOP`、`BEGIN` / `COMMIT`、`RESTART IDENTITY`、`setval` による sequence 同期、データ分布の意図を書くと、あとで読み返しやすいと整理した。

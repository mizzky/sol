### 躓いたポイントと解決策
- 問題: `c.Error(err)` でエラーを積んでも、`ErrorHandler` がレスポンスを書かなければクライアントへ適切なステータス/ボディが返らず、テストが HTTP 200 を返した。
  - 原因: ErrorHandler の実装にレスポンスを書かない経路があった。
  - 解決: `ErrorHandler` が `apperror.ToHTTP` を呼び、`c.JSON(status, gin.H{"error": msg})` を返すよう修正。
- 問題: テスト用の router に `ErrorHandler` を配線していないケースが多数あり、ミドルウェア方式に移行後にテストが失敗した。
  - 原因: 既存テストが個別に `RespondWithError` を期待していたため。
  - 解決: 各テストファイルで `router.Use(middleware.ErrorHandler(apperror.ToHTTP))` を追加して配線。
- 問題: 一部テストで `middleware` / `apperror` の import が不足していた。
  - 解決: 必要な import を追加してコンパイルを通す。
